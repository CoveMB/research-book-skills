# Computational Method Audit

## Audit questions

- Is dataset provenance visible, including source, version, sampling, and exclusions?
- Are preprocessing, feature construction, model or algorithm choice, and evaluation described?
- Are leakage, benchmark contamination, train/test separation, and reproducibility risks addressed?
- Are code, parameters, run logs, seeds, and output pointers available enough to audit?

## Boundaries

Use this reference when a source relies on models, code, scripts, automated extraction, statistics, machine learning, simulation, or computational pipelines.

## Cannot support

- Claims of robust performance without evaluation data.
- Causal claims from predictive models without an identification strategy.
- Manuscript reliance when dataset provenance, leakage checks, or reproducibility material is unavailable.
